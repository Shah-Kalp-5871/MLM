# 1. Introduction

This document explains the **complete business logic** of the Multi-Level Marketing (MLM) system implemented in the Laravel application.

The system is designed to allow users to:

1. Register using referral links
2. Purchase investment packages
3. Earn **ROI income**
4. Earn **level commissions**
5. Achieve **club rewards (voucher income)**
6. Manage funds through a **wallet system**

The system supports a **15-level deep referral network**.

---

# 2. Core Concepts

Before understanding the logic, we must define the core elements.

### User

A person registered in the system.

### Upline

The user who referred another user.

### Downline

A user who joins under another user.

Example:

```
A refers B
B refers C
C refers D
```

Tree Structure

```
A
└── B
    └── C
        └── D
```

Here

A = upline of B
B = upline of C
C = upline of D

---

1. ROI Income

# 4. User Registration Logic

## Registration Methods

Users can register in two ways:

1. Normal registration
2. Referral registration

### Referral Registration Flow

Example:

```
https://website.com/register?ref=USER123
```

Steps:

1. User opens referral link
2. System detects `ref=USER123`
3. System finds the referring user
4. New user is created
5. `upline_id` is stored

Database Example:

| id | name | referral_code | upline_id |
| -- | ---- | ------------- | --------- |
| 1  | A    | A123          | NULL      |
| 2  | B    | B123          | 1         |
| 3  | C    | C123          | 2         |

# 6. ROI Income Logic

ROI = Return on Investment.

ROI is calculated **weekly**.

### ROI Range

Weekly ROI

```
3% – 3.5%
```

Monthly ROI

```
12% – 14%
```

### Example

User invests

```
$500
```

Weekly ROI

```
3% = $15
3.5% = $17.5
```

System calculation example:

```
ROI = deposit_amount × roi_percentage
```

Example:

```
500 × 3% = 15
```

---

# 7. ROI Distribution Logic

ROI is generated automatically using a **cron job / scheduler**.

Laravel scheduler example:

```
weekly ROI distribution
```

Steps:

1. Find all active investments
2. Calculate ROI
3. Add to wallet
4. Trigger level commission

# 12. Commission Distribution Algorithm

When ROI is generated:

```
current_user = ROI receiver
level = 1
```

Loop:

```
while(level <= 15)
    find upline
    calculate commission
    add to wallet
    level++
```
